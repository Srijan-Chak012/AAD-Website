Modern techniques for security

Hash Functions

A cryptographic hash function is an algorithm that takes an arbitrary amount of data input—a credential—and generates a fixed-size output of encrypted text known as a hash value, or simply "hash." That enciphered text can then be saved instead of the password and used to validate the user later.The security of password storage is influenced by certain aspects of cryptographic hash algorithms.

  • Non-reversibility, often known as one-way function.
    A decent hash should make it extremely difficult to deduce the original password from the output or hash.
  • The avalanche effect, or diffusion.
    A change in just one bit of the original password should cause a change in half of the hash's bits. In other words, if a password is slightly modified, the output of enciphered text should change considerably and unexpectedly.
  • Determinism.
    A given password must always produce the same hash value or encrypted text.
  • Collision resistance.
    It should be difficult to uncover two distinct passwords that hash to the same encrypted text.
  • Non-predictable.
    The hash value should be unpredictable based on the password.

Simple cryptographic hashes can slow down attackers, but ultimately attackers will be able to overcome them.

  • Hashed credentials can be quickly "cracked" by attackers using fast hardware.
  • Although good hash algorithms are supposed to be collision-resistant, collisions are hard to totally remove. MD5 and SHA-1 have been shown to have known collisions—that is, they give the same hash value when using different credentials.
  • Rainbow tables are "optimised lookup tables" for reverse-engineering one-way hash algorithms. A rainbow table is essentially a pre-computed set of plaintext strings and hashes. Large rainbow tables are publicly available, and attackers can exploit one of these databases to recover hashed cleartext data.

Symmetric-Key Algos

Symmetric Key Cryptography, also known as Symmetric Encryption, is the use of a secret key for both encryption and decryption. This approach is the inverse of Asymmetric Encryption, which uses one key to encrypt and another to decrypt. Data is changed to a format that cannot be read or inspected by anybody who does not know the secret key used to encrypt it throughout this procedure.

The strength of the random number generator used to generate the secret key determines the effectiveness of this strategy. Symmetric Key Cryptography is commonly used on the Internet nowadays and comprises mostly of two types of algorithms, Block and Stream.

The Advanced Encryption Standard (AES) and the Data Encryption Standard (DES) are two popular encryption algorithms (DES). This type of encryption is often quicker than Asymmetric encryption, but it needs the secret key to be held by both the sender and the recipient of the material.

Asymmetric-Key Algos

The RSA algorithm is an asymmetric encryption algorithm, which means it employs both a public and private key (i.e two different, mathematically linked keys). A public key, as the name implies, is shared publicly, but a private key is private and must not be shared with anybody. The RSA algorithm is named after its developers, Ron Rivest, Adi Shamir, and Leonard Adleman, who created it in 1978.

In the following explanation, the RSA method ensures that the keys are as safe as possible.

Generating Keys

  • Select two large prime numbers, x and y. The prime numbers need to be large so that they will be difficult for someone to figure out.
  • Calculate n = x * y.
  • Calculate the totient function; ϕ(n)=(x−1)(y−1).
  • Select an integer e, such that e is co-prime to ϕ(n) and lies between 1 & ϕ(n). The pair of numbers (n,e) makes up the public key.
  • Calculate d such that e.d = 1 mod ϕ(n). The extended euclidean method can be used to find d. The private key is made up of the pair (n,d).

Encryption

Given a plaintext P in the form of a number, the ciphertext C is computed as follows:
C = Pe mod n

Encryption

Using the private key (n,d), the plaintext can be found using:
P = Cd mod n
